# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Email**: kevin.mauel2+github@gmail.com

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix release**: As soon as practical, depending on severity

## Supported Versions

Only the latest release is supported with security updates.

## Scope

This policy covers the template code itself. Dependencies are monitored via Dependabot and `composer audit` (weekly CI workflow).
